## Interaction Contract

### Simulation Lifecycle State Machine

```
idle --[Start]--> running
running --[Pause]--> paused
running --[Stop Requests]--> stopped_requests
running --[Reset]--> idle
paused --[Resume]--> running
paused --[Reset]--> idle
stopped_requests --[Reset]--> idle
```

**On Start:**
1. Auto-normalize ratios (D-03)
2. Create fresh SimulationEngine with normalized config
3. Create fresh MetricsChartManager with current profile names
4. Create fresh SimulationLoop, wire callbacks
5. Set playback to `running`
6. Disable all parameter fields

**On Reset:**
1. Stop SimulationLoop (cancels rAF)
2. Clear engine/loop refs
3. Reset chart data to empty
4. Set playback to `idle`
5. Re-enable all parameter fields
6. Preserve current config values (user can adjust and re-start)

**On Stop Requests (CTL-03):**
1. Call `engine.stopRequests()` (sets RPS to 0)
2. Set playback to `stopped_requests`
3. Simulation continues running (recovery observable)
4. Speed controls remain active

### Parameter Editing (D-01)

| Playback State | Parameter Fields | Add/Delete Profile | Control Buttons |
|----------------|------------------|--------------------|----------------|
| `idle` | Enabled | Visible | Start only |
| `running` | Disabled | Hidden | Pause, Stop Requests, Reset |
| `paused` | Disabled | Hidden | Resume, Reset |
| `stopped_requests` | Disabled | Hidden | Reset |

### Speed Control Interaction (D-04, D-05)

- Clicking a preset button immediately sets speed and updates slider position
- Dragging slider updates speed in real-time (no debounce needed)
- If current speed exactly matches a preset, that preset button shows active style
- Speed control disabled (opacity-50) when playback is `idle` or `paused`

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | "Start Simulation" (idle state) |
| Resume CTA | "Resume" (paused state) |
| Stop requests CTA | "Stop Requests" (running state) |
| Pause CTA | "Pause" (running state) |
| Reset CTA | "Reset" (running/paused/stopped_requests state) |
| Empty state (Status) | Elapsed: "--", 503s: "--", Ready: "--" |
| Empty state (no profiles) | Not applicable -- minimum 1 profile enforced |
| Error state (invalid input) | Red border on field, no toast/modal. Prevent start if podCount < 1 or workersPerPod < 1 or no profiles |
| Start button disabled tooltip | "Fix invalid parameters to start" (shown when validation fails) |
| Ratio normalization hint | "Ratios auto-normalize to 100%" (text below Request Profiles section, 12px, text-secondary) |
| Destructive: Reset | No confirmation dialog. Reset is low-cost (user can re-start with same params). Immediate action. |
| Section headings | "Cluster", "Traffic", "Request Profiles", "Liveness Probe", "Readiness Probe", "Pod" |
| Speed display | "{value}x" (e.g., "1x", "10.5x", "100x") |
| Elapsed time format | "00:00" (mm:ss) or "0:00:00" (h:mm:ss) |
