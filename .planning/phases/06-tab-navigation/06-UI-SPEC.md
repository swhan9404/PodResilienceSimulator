### BackgroundActivityBadge

Small green dot indicator on the "Simulator" tab label when simulation is running and user is on the Optimizer tab.

| Property | Value |
|----------|-------|
| Shape | Circle |
| Size | 8px x 8px (`w-2 h-2`) |
| Color | `#22C55E` (success green) |
| Position | Inline after "Simulator" text, `ml-2` (8px gap) |
| Border radius | `rounded-full` |
| Visibility | Shown when `playback === 'running'` AND `activeTab === 'optimizer'` |
| Animation | None (static dot, no pulse) |

Source: D-06, D-07 from CONTEXT.md. Color matches existing success green from `SummaryCards.tsx`.

### TabPanel Container (keepMounted)

Both tab panels are always mounted. Only the active one is visible.

| Property | Active Panel | Inactive Panel |
|----------|-------------|----------------|
| CSS | `visibility: visible; position: relative; height: auto;` | `visibility: hidden; position: absolute; top: 0; left: 0; width: 100%; height: 0; overflow: hidden;` |
| DOM presence | Always mounted | Always mounted |
| Rendering | Normal flow | Hidden, no layout space |
| ARIA | `aria-hidden="false"`, `role="tabpanel"` | `aria-hidden="true"`, `role="tabpanel"` |

Each panel contains a full `flex` layout: sidebar (300px) + main content area.

**Simulator panel:** Wraps the existing `ControlPanel` + main visualization area exactly as current `App.tsx`.

**Optimizer panel:** Placeholder content for Phase 7.

Source: D-08, D-09 from CONTEXT.md. `visibility: hidden` + `position: absolute` preserves Canvas refs, RAF loop, and uPlot instances. `height: 0; overflow: hidden` prevents the hidden panel from taking scroll space.

## Interaction Contract

### Tab Switching

| Interaction | Behavior |
|-------------|----------|
| Click inactive tab | `activeTab` state updates in Zustand store; CSS classes swap on panels; transition is instant (no animation, no loading state) |
| Keyboard: Tab + Enter/Space | Same as click -- tabs are focusable and activate on Enter or Space |
| Keyboard: Arrow Left/Right | Moves focus between tabs (standard ARIA tab pattern) |
| Browser back button | No effect (no URL change, no history entries) |

### State Preservation

| Scenario | Expected Behavior |
|----------|-------------------|
| Start simulation -> switch to Optimizer -> switch back | Simulation still running, RAF loop intact, Canvas renders current state, charts updating |
| Configure optimizer inputs -> switch to Simulator -> switch back | Optimizer inputs preserved (Phase 7 concern but architecture supports it) |
| Resize window while on inactive tab | Active tab resizes normally; inactive tab resizes on next activation (Canvas `onResize` fires when visible) |

Source: ROADMAP.md Phase 6 success criteria #2, #3; D-08 from CONTEXT.md.

## Layout Specification

```
+------------------------------------------------------------------+
| [Slow Request Simulator]   [Simulator]  [Optimizer]              | <- TabBar (48px)
+------------------------------------------------------------------+
| Sidebar   |  Main Content Area                                   |
| (300px)   |  (flex-1)                                            |
|           |                                                      |
|           |                                                      |
|           |                                                      |
|           |                                                      |
+------------------------------------------------------------------+
```

- Tab bar spans full width at the top
- Below tab bar: active panel fills remaining viewport height
- Each panel has its own sidebar (300px) + main area (flex-1)
- `min-w-[1280px]` constraint maintained on panel containers
- Tab bar itself has no min-width constraint (stretches naturally)

Source: Existing `App.tsx` layout pattern, D-01 through D-05 from CONTEXT.md.

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Tab bar role | `role="tablist"` on container |
| Tab role | `role="tab"` on each tab button |
| Panel role | `role="tabpanel"` on each panel |
| Active state | `aria-selected="true"` on active tab |
| Panel linkage | `aria-controls="{panel-id}"` on tab, `aria-labelledby="{tab-id}"` on panel |
| Hidden panel | `aria-hidden="true"` on inactive panel |
| Keyboard | Arrow keys move focus between tabs; Enter/Space activates; Tab key moves to panel content |
| Focus visible | Browser default focus ring (no custom styling) |
