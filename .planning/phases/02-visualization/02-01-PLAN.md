**Create `src/visualization/PodRenderer.ts`:**

A plain TypeScript class that draws PodSnapshot[] onto a Canvas 2D context. Key implementation details:

Constructor takes `ctx: CanvasRenderingContext2D` and stores it. Has a `draw(pods: PodSnapshot[], canvasWidth: number, canvasHeight: number, isDark: boolean)` method.

Layout calculation (per D-01, UI-SPEC pod grid layout):
- Column count formula from UI-SPEC:
  - 1-3 pods: cols = podCount
  - 4 pods: cols = 2
  - 5-6 pods: cols = 3
  - 7-9 pods: cols = 3
  - 10-12 pods: cols = 4
  - 13-16 pods: cols = 4
  - 17-20 pods: cols = 5
- Implementation: use `Math.ceil(Math.sqrt(podCount))` but cap at these specific values
- Gap between pods: 16px (md token from UI-SPEC)
- Worker cell size: 20x20px, gap 4px (xs token)
- Pod card internal padding: 8px (sm token)
- Pod label height: 16px
- Backlog label height: 12px
- Probe row height: 12px
- Pod border width: 3px, border radius: 4px

For each pod card, draw in this order:
1. Pod card background fill (rounded rect) with state-colored border (3px):
   - `PodState.READY` -> `#22C55E`
   - `PodState.NOT_READY` -> `#F59E0B`
   - `PodState.RESTARTING` -> `#EF4444`
2. Pod label: `"Pod {id}"` at 12px semibold, top-left inside padding. Font: `'600 12px system-ui, sans-serif'`
3. Worker grid: wrap at `Math.ceil(Math.sqrt(workersPerPod))` columns. Each cell 20x20px with 4px gap.
   - Idle (busy=false): fill with theme workerIdle color
   - Busy (busy=true): fill with `worker.profileColor` (falls back to `#888888` if null)
4. Backlog text: `"BL: {backlogSize}/{backlogMax}"` at 10px regular. Font: `'400 10px system-ui, sans-serif'`
5. Probe rows:
   - `"L: "` followed by liveness history glyphs: `+` for true (green), `x` for false (red)
   - `"R: "` followed by readiness history glyphs: `+` for true (green), `x` for false (red)
   - Font: `'400 10px system-ui, sans-serif'`
   - Use `fillText` for each glyph with appropriate color from `getThemeColors(isDark)`

The `draw()` method must:
1. Call `ctx.clearRect(0, 0, canvasWidth, canvasHeight)` first
2. Fill entire canvas with theme canvas background color
3. Calculate layout (can be a separate `calculateLayout` method that is also exported for testing)
4. Center the grid within the canvas
5. Draw each pod card

Export `calculateLayout` as a standalone function (not just a method) so it can be unit tested:
```typescript
export function calculateLayout(podCount: number, workersPerPod: number, canvasWidth: number, canvasHeight: number): PodLayout
```

**Create `src/visualization/PodCanvas.tsx`:**

React component that:
1. Creates a `<canvas>` element via `useRef<HTMLCanvasElement>`
2. On mount (useEffect), sets up DPI-aware canvas:
   ```typescript
   const rect = canvas.getBoundingClientRect();
   const dpr = window.devicePixelRatio || 1;
   canvas.width = rect.width * dpr;
   canvas.height = rect.height * dpr;
   const ctx = canvas.getContext('2d')!;
   ctx.scale(dpr, dpr);
   ```
3. Creates a `PodRenderer` instance stored in a ref
4. Sets up a `ResizeObserver` on the canvas that:
   - Compares new dimensions with cached previous dimensions (to avoid infinite loop per Pitfall 4)
   - If changed: re-runs DPI setup, tells renderer about new size
5. Exposes a method (via ref or callback) to trigger drawing: `rendererRef.current.draw(pods, width, height, isDark)`
6. Detects dark mode via `window.matchMedia('(prefers-color-scheme: dark)')` and listens for changes
7. Props: accepts an `onRendererReady` callback or uses `useImperativeHandle` to expose `draw` method. For Phase 2, the simplest approach is to expose the renderer ref so SimulationLoop can call draw directly.

Actually, for cleaner integration with SimulationLoop (Plan 03), PodCanvas should:
- Accept a `rendererRef: React.MutableRefObject<PodRenderer | null>` prop
- On mount, create PodRenderer and assign to `rendererRef.current`
- On unmount, set `rendererRef.current = null`

Canvas element styling: `className="w-full"` with explicit `style={{ height: '300px' }}` (will be adjusted by SimulationLoop based on pod count). The canvas CSS dimensions define the drawing area; the DPI setup scales pixel buffer for retina.
