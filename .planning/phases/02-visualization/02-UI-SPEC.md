# Phase 2 -- UI Design Contract

> Visual and interaction contract for the visualization phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (Phase 2 is Canvas + uPlot, no DOM components) |
| Icon library | none (probe results use Unicode glyphs: U+2713 / U+2717) |
| Font | system-ui, 'Segoe UI', Roboto, sans-serif (inherited from project index.css) |

**Rationale:** Phase 2 renders entirely via Canvas 2D API and uPlot. There are zero DOM-based interactive controls (buttons, inputs, modals). shadcn/radix are not applicable. Phase 3 will introduce DOM controls and may revisit component library choice at that time.

### Pod Canvas Grid Layout (per D-01 -- Claude's discretion)

Column count formula: `Math.ceil(Math.sqrt(podCount))`

| Pod Count | Columns | Rows |
|-----------|---------|------|
| 1-3 | podCount | 1 |
| 4 | 2 | 2 |
| 5-6 | 3 | 2 |
| 7-9 | 3 | 3 |
| 10-12 | 4 | 3 |
| 13-16 | 4 | 4 |
| 17-20 | 5 | 4 |

Gap between pod cards: 16px (md token).

### Pod Card Internal Layout

```
+-------------------------------+
| Pod 0                  (label)|  <- 12px semibold, top-left
|-------------------------------|
| [W][W][W][W]    (worker grid) |  <- 20x20 cells, 4px gap
| [W][W][W][W]                  |
|-------------------------------|
| BL: 3/10        (backlog text)|  <- 11px regular, left-aligned
|-------------------------------|
| L: +++x  R: +++x (probes)    |  <- 10px, L=liveness R=readiness
+-------------------------------+
  ^--- 3px border, color = pod state
```

Worker grid wraps at: `Math.ceil(Math.sqrt(workersPerPod))` columns per pod.

### Chart Grid Layout (per D-07)

2x2 grid with 16px gap between charts. Each chart fills 50% of available width minus gap.

| Position | Chart | Y-axis Label | Y Range |
|----------|-------|--------------|---------|
| Top-left | Worker Usage | % | 0-100 |
| Top-right | Ready Pods | count | 0-podCount |
| Bottom-left | 503 Rate | % | 0-100 |
| Bottom-right | Response Time | ms | auto-scale |

Chart height: 200px each (400px total for 2 rows + gap).
Time window: 60 seconds of simulation time, scrolling (per D-06).

## Rendering Contract

### Performance Targets (per D-09, D-10, D-11)

| Metric | Target | Measurement |
|--------|--------|-------------|
| Canvas frame rate | 60fps | requestAnimationFrame callback consistent |
| Canvas draw time | < 4ms per frame | performance.now() around draw call |
| Chart update rate | 1Hz (1 update per simulated second) | Throttled by MetricsSample interval |
| Chart setData time | < 2ms per call | performance.now() around uPlot.setData() |
| Engine step time | < 8ms per frame at 100x | performance.now() around engine.step() |
| Total frame budget | < 16ms (at 60fps) | Sum of engine.step + canvas.draw + chart overhead |

### Canvas Rendering Rules

1. Full redraw every frame (clearRect + draw all pods). No dirty-region tracking.
2. DPI-aware: canvas pixel dimensions = CSS dimensions x devicePixelRatio. Context scaled accordingly. (Pitfall #12)
3. Pod card positions computed once on layout change (pod count change or resize), cached until next change.
4. Worker cell fill: idle = gray, busy = `WorkerSnapshot.profileColor`, progress not visually represented (cell is fully colored when busy).
5. Pod border color derived from `PodSnapshot.state` using the semantic color map above.

### Chart Rendering Rules

1. uPlot instances created once on mount, destroyed on unmount.
2. Data updates via `uPlot.setData(data)`, never instance recreation.
3. Resize via `ResizeObserver` on container element, calling `uPlot.setSize()`.
4. Time window: show last 60 simulated seconds. Older data kept in buffer but not displayed.
5. uplot-react wrapper handles lifecycle (per Pitfall #16).

### Auto-Run Demo Config (per D-11)

Phase 2 starts simulation immediately with hardcoded defaults:

| Parameter | Value |
|-----------|-------|
| podCount | 5 |
| workersPerPod | 4 |
| maxBacklogPerPod | 10 |
| rps | 50 |
| speed | 1x |
| requestProfiles | normal (200ms, 70%, #3B82F6), slow (5000ms, 30%, #F97316) |
| livenessProbe | period=10s, timeout=1s, failureThreshold=3, successThreshold=1 |
| readinessProbe | period=5s, timeout=1s, failureThreshold=3, successThreshold=1 |
| initializeTimeMs | 30000 (30s) |
