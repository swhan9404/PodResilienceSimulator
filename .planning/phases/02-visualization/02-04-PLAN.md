Fix 4 TypeScript errors that block `npm run build`:

**File: src/simulation/engine.test.ts**

1. Line 3 — TS1484 (verbatimModuleSyntax): `SimulationConfig` is a type-only export and must use `import type` under verbatimModuleSyntax. `PodState` is an enum (runtime value) and must stay as a regular import. Change line 3 from:
   ```
   import { SimulationConfig, PodState } from './types';
   ```
   to two separate imports:
   ```
   import type { SimulationConfig } from './types';
   import { PodState } from './types';
   ```

2. Lines 89-90 — TS6133: `engine1` and `engine2` are declared but never read. These variables are created on lines 89-90 but the test immediately creates new `e1` and `e2` variables on lines 99-100 using a different config. The `engine1`/`engine2` variables serve no purpose. Remove lines 89-90 entirely (the two lines that read `const engine1 = new SimulationEngine(makeConfig({ seed: 42 }));` and `const engine2 = new SimulationEngine(makeConfig({ seed: 99 }));`).

**File: src/simulation/pod.test.ts**

3. Line 3 — TS6133: `ProbeResult` is imported but never used anywhere in the test file. Remove line 3 entirely (the line that reads `import type { ProbeResult } from './pod';`).

Do NOT change any other code. Do NOT modify test logic. Do NOT reformat or reorganize imports beyond these specific fixes.
