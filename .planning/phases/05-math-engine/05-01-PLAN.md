    **Step 2: Create `src/optimizer/queuing.ts`** (per D-10: pure functions, no class)

    Implement two exported functions:

    **`computeEffectiveWorkers`** (per D-01, D-02, D-03):
    - Parameters: `workersPerPod: number`, `probes: ProbeParams`
    - Compute: `livenessRate = 1 / probes.livenessProbe.periodSeconds`
    - Compute: `readinessRate = 1 / probes.readinessProbe.periodSeconds`
    - Compute: `probeDutyCycle = (livenessRate + readinessRate) * 0.001` (D-02: 1ms = 0.001s)
    - Return: `Math.max(workersPerPod - probeDutyCycle, 0.01)` (clamp to prevent negative/zero)

    **`computeMMcK`** (iterative ratio accumulation, NO factorials):
    - Parameters: `lambda: number, mu: number, c: number, K: number`
    - Compute `r = lambda / mu` (offered load in Erlangs)
    - Compute `rho = lambda / (c * mu)` (per-server utilization)
    - **Guard**: if `c <= 0 || K <= 0 || mu <= 0`, return `{ rho: Infinity, pBlock: 1, wq: Infinity, lq: K }`
    - **Guard**: if `lambda <= 0`, return `{ rho: 0, pBlock: 0, wq: 0, lq: 0 }`
    - Build state probability ratios iteratively (NO factorials):
      - `sumRatios = 1.0`, `ratio = 1.0` (for n=0)
      - For n=1..c: `ratio *= r / n`, `sumRatios += ratio`
      - Save `ratioAtC = ratio` (this is r^c / c!)
      - For n=c+1..K: `ratio *= r / c` (note: divide by c, not n), `sumRatios += ratio`
    - Compute `p0 = 1 / sumRatios`
    - Compute `pK = ratio * p0` (blocking probability)
    - Compute Lq (mean queue length):
      - If `Math.abs(rho - 1.0) < 1e-10`: use special-case formula `lq = ratioAtC * p0 * (K - c) * (K - c + 1) / 2`
      - Else: `lq = (ratioAtC * rho * p0) / ((1 - rho) ** 2) * (1 - Math.pow(rho, K - c) - (K - c) * Math.pow(rho, K - c) * (1 - rho))`
    - Compute effective arrival rate: `lambdaEff = lambda * (1 - pK)`
    - Compute `wq = lambdaEff > 0 ? lq / lambdaEff : 0` (Little's law)
    - Return `{ rho, pBlock: pK, wq, lq }`
